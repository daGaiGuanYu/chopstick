## 功能
+ 启动一个 http 服务器
+ 处理 http 请求的 router
+ 处理客户的异常操作
+ 阻止代码错误引发的崩溃

## 定位
前后端分离后的“后端”的工具集（基于 node），所以：
+ 用来写接口
+ 不提供静态资源托管（以后也不会），推荐使用 nginx 等优秀项目

## 优点
+ 简单：源码简单、使用简单、文档简单
+ 单纯
+ AOP
+ 中文本位

## 原则
